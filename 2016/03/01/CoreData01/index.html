CoreData简介

概述

Core Data是iOS5之后才出现的一个框架，它提供了对象-关系映射(ORM)的功能，即能够将OC对象转化成数据，保存在SQLite数据库文件中，也能够将保存在数据库中的数据还原成OC对象，他与 sqlite 对比最大的优点莫过于支持对象的存储了， CoreData是apple自己封装的数据库操作的一个框架, 是一种面向对象的方式来操作数据的框架, 底层是基于SQLite的。

为什么要用Core Data

  • Core Data是对底层存储的一次封装。封装了以后就变成ORM的框架。这样就变成操作对象。Core Data自己去进行数据的保存。
  • 使用Core Data而不是FMDB，让整个程序架构更加的面向对象。
  • Core Data仅仅使用了Objective-C和Core Foundation，你不必去加入一些第三方的库。
  • Core Data是Apple的原生技术。每年的WWDC都能看到新特性的加入和讲授。
  • Core Data支持iCloud。而使用iCloud的App。Apple推荐的可能性增加。
  • 有了iCloud为以后Apple的全平台数据共享打下基础

CoreData显著特性

  • 变更管理（撤销与重做）
  • 关系relationship
  • 数据模型的版本管理及迁移
  • 通过batching及faulting高效地获取数据
  • 通过谓词高效地过滤数据
  • 数据一致性及数据验证

常用类

  • NSPersistentStore：持久化存储区
  • NSPersistentStoreCoordinator: 持久化存储协调器，用来关联类和对应的数据库文件
  • NSManagedContext: 托管对象上下文, 用来操作数据库的增删改查
  • NSManagedObjectModel: 托管对象模型，对应于所有实体类的描述文件, 文件里面包含很多类的属性和关系的描述
  • NSManagedObject: 托管对象，对应于一个实体类, 所有的CoreData操作的实体类都是它的子类

托管对象模型

托管对象模型是一种数据结构，可以通过File -> New -> iOS -> CoreData -> Data Model创建

实体

托管对象模型由一系列实体描述对象构成，这种对象就叫做实体。实体就好比一把模型切割刀，用于创建托管对象。托管对象模型可以拥有一个或多个实体，在创建托管对象之前，要先把实体设计好。

在设计数据库中数据表时，需要完成以下几步：

  • 配置数据表名称（table name）
  • 配置字段并未每个字段设定“数据类型”

在设计实体时，需要做的是：

  • 配置实体名称
  • 配置属性，并为每个属性设定数据类型
  • 根据实体来配置NSManagedObject的子类（可选）

通过Xcode添加一个名为Item的实体

选定Model.xcdatamodeld -> 点击Add Entity -> 把心实体名称改为Item

并为它添加属性

选定Item -> 点击Add Attribute
